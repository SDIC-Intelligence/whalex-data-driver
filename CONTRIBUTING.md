# Contributing to whalex-data-driver

欢迎来到 **whalex-data-driver**！🎉  
这是一个用于统一抽象不同大数据组件、云组件的 **数据操作驱动层**。  
本文档为贡献指南，帮助新成员顺利参与项目。

如果发现内容有误或不完整，欢迎提出 Issue 或 PR。

---

## 🚀 参与贡献前

### 角色说明

在 whalex-data-driver 中，我们鼓励不同角色参与：

- **User（用户）**：使用项目，发现问题或提供使用反馈  
- **Contributor（贡献者）**：提交 Bug 修复、文档改进或功能增强  
- **Committer（核心贡献者）**：有权合并 PR，参与项目决策  
- **PMC（项目管理委员会）**：项目治理角色  

你的角色会随着贡献累积逐步提升。

### Code of Conduct

请务必阅读并遵守我们的 [行为规范](CODE_OF_CONDUCT.md)。

---

## 📝 如何贡献

### 1. 提交 Issue

在开始贡献前，可以先提交 Issue，类型包括：

- 🐞 Bug 报告  
- 💡 新功能建议  
- 📚 文档改进  

建议包含：

- 系统环境（JDK、组件版本等）  
- 最小复现步骤  
- 错误日志（如适用）  
- 期望行为  

> **注意**：任何 PR 必须关联有效 Issue，否则会被拒绝。

---

### 2. 开始贡献

1. Fork 本仓库  
2. 克隆到本地  
3. 创建新分支（如 `feature/<name>` 或 `fix/<issue-id>`）  
4. 保持分支与远程 develop 分支同步  
5. 提交代码，信息简洁明确  
6. Push 到你的 fork  
7. 创建 Pull Request 到 `develop` 分支  

**PR 注意事项：**

- 使用 PR 模板  
- 保证关联 Issue  
- PR 尽量不太大，复杂功能可拆分  
- 大改动请附详细设计文档  

---

## 👀 代码审核

- PR 会分配至少一名 reviewer  
- 遵循可读性、优雅、可测试、可维护原则  
- 必须修复 review 建议后才能合并  

---

## 🔀 分支策略

- **main**：稳定分支（发布版本）  
- **develop**：开发分支  
- **feature/***：新功能  
- **fix/***：Bug 修复  

---

## 🌟 晋升路径

成为 Committer 的建议步骤：

1. 提交至少 8 个非 trivial PR  
2. 至少 3 个不同贡献者 review  
3. 展示协作能力、代码理解与设计能力  
4. 提交提名申请（包括姓名、Git 链接、理由）  
5. 2 名 Committer 支持，5 个工作日无异议即成为 Committer
